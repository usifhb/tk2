# نشر المتجر والوصول إلى Super Admin

## بعد رفع الموقع (مثلاً على Laravel Cloud)

### 1. التأكد من الإعدادات
- ضبط متغيرات البيئة: `APP_URL`, `APP_KEY`, `DB_*`, `ADMIN_DIR` (اختياري، الافتراضي `admin`).
- تشغيل الـ migrations إن لم تكن شغّلت تلقائياً:
  ```bash
  php artisan migrate --force
  ```

### 2. إنشاء حساب Super Admin (أول مسؤول)
لأن معالج التثبيت معطّل، أنشئ أول مستخدم أدمن من الطرفية:

```bash
php artisan cms:user:create
```

سيُطلب منك:
- **First name** – الاسم الأول
- **Last name** – اسم العائلة
- **Email address** – البريد (يُستخدم لتسجيل الدخول أو استعادة كلمة المرور)
- **Username** – اسم المستخدم (يُستخدم لتسجيل الدخول)
- **Password** – كلمة المرور (6 أحرف على الأقل)

هذا الحساب يكون **Super User** (صلاحيات كاملة).

### 3. الدخول إلى لوحة الإدارة
- عنوان الدخول: **`https://دومينك.com/admin`** (أو القيمة التي ضبطتها في `ADMIN_DIR`).
- سجّل الدخول بالـ **Username** أو **Email** وكلمة المرور التي أدخلتها في الأمر أعلاه.

### 4. إنشاء super admin إضافي لاحقاً
- من لوحة الإدارة: **Users** → **Create** ثم إنشاء مستخدم وعطله صلاحية **Super User**.
- أو من الطرفية مرة أخرى: `php artisan cms:user:create` (ينشئ super user جديد).

---

**ملاحظة:** غيّر `ADMIN_DIR` في `.env` (مثلاً إلى اسم غير واضح) لتقليل محاولات الدخول على `/admin`.
